# New 3D Modules Subpage

This page allows administrators to upload and manage new 3D industrial projects and modules with complete metadata, 2D sketches, and supporting documentation.

Supported 3D file formats:

- **Primary format:** GLB (recommended for web performance, compression, and material/texture support)
- **Additional formats:** GLTF, STL, OBJ, FBX

File upload workflow:

1. Admin selects or drags a 3D model file (GLB, GLTF, STL, OBJ, or FBX).
2. System validates file integrity and size constraints.
3. Admin enters required metadata (see below).
4. Admin uploads 2D sketches, technical drawings, or documentation.
5. Admin reviews and submits for publishing.
6. Module is queued for approval before appearing on the public portfolio.

Required fields:

- Module title (Arabic and English)
- Project description (Arabic and English) — purpose, use case, mechanical details
- Category/industry classification
- Designer/engineer name
- Upload date
- 3D model file (GLB primary)
- Technical drawings or 2D sketches (optional but recommended)
- Materials and specifications (optional)
- Performance notes or special features (optional)

Admin notes:

- All content must comply with the project's [security and content policies](../Constraints.md) (see Rule #5).
- Uploaded modules are previewed using the Three.js viewer before publication.
- Admins can edit, delete, or unpublish modules after creation.
- Maintain an audit trail of all uploads and modifications.
